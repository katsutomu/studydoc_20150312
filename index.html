一言で言うと

C#でアプリが作れるクロスプラットフォームエンジン

そもそもクロスプラットフォームエンジンって？

  • ひとつのソースで複数の環境で動くアプリが作れる
  • アプリの開発したことなくても作れる
  • UIは各プラットフォーム間で同じになる

クロスプラットフォームで目指すところ

  • 短納期・多デバイス

問題点

  • UIが同じになる
  • 学習コスト高い
  • 重い！

そこでXamarin

良いところ

  • UIを分けられる
  • 既存開発の資産を活用できる
  • ライブラリが薄い

できること

  • C＃で各OSのAPIを叩く
  • ビジネスロッジクの共有
  • 過去の資産の活用

API

実行モデル

できないこと

  • Write once, Run Anywhere
